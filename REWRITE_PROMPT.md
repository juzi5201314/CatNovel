# Author 项目重写参考（功能总结 + 可复制提示词）

## 一、项目功能总结（基于当前仓库）

这个项目本质是一个 **AI 辅助小说创作工作台（Web 为主）**，核心由四条主线组成：

1. 写作编辑主线  
基于 TipTap 的富文本编辑器，支持标题/列表/代码块/高亮/上下标/公式（KaTeX）等格式。  
编辑区做了 Word 风格的虚拟分页，支持字体、字号、行距、页边距调整，实时显示字数/页数等统计。  
集成 Ghost Text 交互：AI 流式生成后先以“幽灵文本”展示，用户可接受/拒绝/重生。

2. AI 创作主线  
支持多供应商（OpenAI 兼容、DeepSeek、智谱、Gemini 等）的流式生成。  
功能覆盖续写/改写/润色/扩写与自由对话。  
有上下文引擎：会自动拼接章节内容、设定集、写作模式信息，并按 token 预算裁剪。  
支持 embedding + 相似度检索（RAG）自动召回设定条目。

3. 设定集与项目管理主线  
设定集是树形结构（角色、地点、物品、规则等），支持多作品切换。  
提供设定导入导出、冲突处理、写作模式切换（网文/文学/剧本）。  
支持章节管理（增删改、重命名、导出）、项目级导入导出、快照管理（手动/自动、恢复/删除）。

4. 产品体验主线  
三栏布局（左侧章节与项目操作，中间编辑器，右侧 AI）。  
支持主题切换、国际化（中文/英文/俄文）、引导层、欢迎弹窗、帮助面板、更新提示、Toast 通知。  
默认本地优先数据策略（当前实现主要使用 IndexedDB + localStorage）。

另外，当前仓库还包含 Electron 打包与更新逻辑，但你已明确 **后续重写不需要 Electron**。
